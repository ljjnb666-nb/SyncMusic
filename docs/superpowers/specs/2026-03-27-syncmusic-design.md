# SyncMusic 设计文档

## 项目概述

- **项目名称**: SyncMusic
- **功能**: 两个用户异地同步播放QQ音乐/网易云/酷狗音乐
- **目标用户**: 两人一起听歌的需求
- **开发模式**: 本地开发测试 → 最终 Docker 部署

## 核心功能

### 1. 音乐解析模块

| 功能 | 描述 |
|------|------|
| 输入链接解析 | 支持QQ音乐、网易云音乐、酷狗音乐的歌曲/歌单链接 |
| 获取音频URL | 获取真实可播放的音频流地址 |
| 歌曲信息 | 歌名、歌手、封面、时长 |

**技术实现**:
- 后端调用各平台公开API获取歌曲信息
- 解析音频流URL（参考各平台网页版播放器原理）

### 2. 同步播放模块

| 功能 | 描述 |
|------|------|
| 播放状态同步 | 播放/暂停/停止 实时同步 |
| 播放进度同步 | 当前时间戳同步，误差<500ms |
| 切歌同步 | 切换歌曲时双方同时切换 |
| 同步控制 | 任意一方操作都会同步到另一方 |

**技术实现**:
- WebSocket (Socket.IO) 实现实时通信
- 房主模式：创建房间，一方控制，另一方跟随

### 3. 歌单管理模块

| 功能 | 描述 |
|------|------|
| 创建歌单 | 本地创建播放列表 |
| 导入歌单 | 导入QQ音乐/网易云/酷狗音乐歌单链接 |
| 分享歌单 | 生成邀请链接，另一人加入同一房间 |

### 4. 用户模块

| 功能 | 描述 |
|------|------|
| 配对机制 | 两人通过房间号配对 |
| 房主/听众 | 房主控制播放，听众自动跟随 |

## 数据流

### 同步流程

```
用户A(房主) 操作播放
       ↓
前端发送 "play" 事件 + 当前时间戳
       ↓
Socket.IO 广播到同一房间的所有用户
       ↓
用户B 前端接收事件，更新播放状态
```

### 音乐解析流程

```
用户输入歌曲链接
       ↓
后端解析链接类型(QQ/网易云/酷狗)
       ↓
调用对应平台API获取歌曲信息
       ↓
返回音频URL + 歌曲metadata
       ↓
前端使用音频URL进行播放
```

REST Endpoints:

Socket.io Events:
- Client emits: room:join { roomId, username }
- Server emits: room:state { full room object }
- Server emits: room:join { userId, username }
- Server emits: room:leave { userId }
- Server emits: room:host { hostId }

Room State Schema:
```javascript
{
  id: string,
  name: string,
  hostId: string,
  participants: [{ id, username, joinedAt, isHost }],
  isPlaying: boolean,
  currentTrack: null,
  position: 0
}
```

  <action>
Create frontend/src/stores/room.js with Pinia store:

1. Import { defineStore } from 'pinia'
2. Import socket from '../socket/client' (lazy import to avoid circular)
3. Define store 'room' with:

State:
- `id: null` - room ID
- `name: ''` - room name
- `hostId: null` - current host's user ID
- `participants: []` - array of { id, username, joinedAt, isHost }
- `isPlaying: false`
- `currentTrack: null`
- `position: 0`
- `sessionId: null` - THIS user's session ID (from localStorage)
- `isConnected: false` - socket connection status

Getters:
- `isHost` - returns true if sessionId === hostId
- `participantCount` - returns participants.length
- `participantList` - returns participants array sorted by joinedAt
- `hostParticipant` - returns participant object where isHost === true

Actions:
- `setRoom(room)` - sets all room state from server room:state event
- `setSessionId(id)` - stores this user's session ID
- `addParticipant({ userId, username })` - adds to participants array with joinedAt: Date.now(), isHost: false
- `removeParticipant(userId)` - filters out participant by id
- `setHost(hostId)` - updates hostId and sets isHost on correct participant
- `clearRoom()` - resets all room state to initial values
- `updateParticipantUsername(userId, username)` - updates username for participant

4. Handle socket events in a separate `initSocket()` method called once:
   - socket.on('room:state', (room) => setRoom(room))
   - socket.on('room:join', ({ userId, username }) => addParticipant({ userId, username }))
   - socket.on('room:leave', ({ userId }) => removeParticipant(userId))
   - socket.on('room:host', ({ hostId }) => setHost(hostId))
</action>

  </verify>

    - setHost updates both hostId and participant.isHost flags
    - Socket events wired to store actions
